This file defines a map between this ISA and x86-style instructions, used for
reference. It also includes the side effects of the instructions.
x86 was chosen for familiarity; this ISA does not compile to x86 or anything
like that (yet).
-------------------------------------------------------------------------------
R refers to a register index.
I refers to an immediate word.
M refers to an immediate memory address.
I4, I8, and I12 refer to sized immediates, for bit budgeting.

X, in a mapping, means that this ISA can use an immediate here, but x86 must
use r16, m16, or r/m16 in place.

A right-pointing arrow '->' at the start of a mapped instruction means that if
this was x86, we would need the defined fixed register for an input.
A right-pointing arrow '->' at the end of a mapped instruction means that if
this was x86, we would derive the result of intended instruction from the
defined fixed register.
This ISA does not use implicit registers, but the mapping is explicit.

'(2)' at the end of an effect means its operation follows 2's compliment. If
it is not present, the operation can be treated as unsigned, or disregarding
sign.
'(Z)' at the end of an effect means its operation zero-extends the result.
'(S)' at the end of an effect means its operation sign-extends the result,
or otherwise uses the sign bit (eg Shift Arithmetic Right fills with sign).

Square brackets denote which flags an effect sets, if any - unspecified flags
are ignored, but places assigned to a flag which are 0 shall be cleared.
Letters representing flags and their assigned places include:
        - Z for zero flag,            place 1
        - N for negative (sign) flag, place 2
        - C for carry flag,           place 3
        - V for overflow flag,        place 4
